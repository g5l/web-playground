Detecting Classes in Source Files

Tailwind tokenizes source as plain text — not code. A class must exist as a complete, unbroken string at build time, or be force-generated via @source inline().

The rule:  'bg-' + color + '-500' is three tokens, none matching a class.  'bg-red-500' is one token that does.  Click each swatch and watch "css:" flip.

1 · Dynamic (broken)

—

// split across tokens at build time
const cls = 'bg-' + c + '-700'
el.className = cls  // class exists,
                    // CSS does not

2 · Lookup table (works)

—

const map = {
  red:   'bg-red-500',   // ✓ literal
  blue:  'bg-blue-500',  // ✓ literal
  green: 'bg-green-500', // ✓ literal
  amber: 'bg-amber-500', // ✓ literal
}

3 · @source inline (works)

—

/* tailwind.css — brace expansion */
@source inline(
  "bg-{fuchsia,lime,sky,pink}-500"
);
// same dynamic JS as demo 1,
// but Tailwind pre-generated them
